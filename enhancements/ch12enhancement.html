Ch. 12 Enhancement

Purpose

Each enhancement provides an opportunity to solve problems and apply the concepts covered in the current and previous topics.

Assumptions

This enhancement requires your use of the chapter 7 enhancement as the starting point. You must have completed the chapter 7 enhancement so that you are familiar with the Future Value application and how it operates.

Make a Copy

Prior to beginning the enhancement, make a copy of the chapter 7 enhancement folder and paste it to a homework area in your site. Rename the pasted folder to represent the chapter 12 enhancement. Do all work on the enhancement in this location. Do NOT overwrite your chapter 7 enhancement doing this one. Keep them separate.

Tasks

You will rework the Future Value application by implementing a persistent session and storing values in it. The specific functionality to be built is described below:

  1. Insure that the controller and all views have access to the session.
  2. Create a persistent session that will last for two weeks.
  3. Store the values from the Future Value input form into the session and populate the form with the values from the session when the page containing the form is loaded if there are values in the session, otherwise make the form inputs blank.
  4. The calculation capability of the application remains unchanged.

Submission

  • Build and test your code in the local development environment.
  • Check your work to insure that it is operating correctly.
  • When satisfied that the code is operational, upload the enhancement folder to the remote production server.
  • Create a link from your exercises page to the index page in the enhancement folder and test that it works.
  • Fully test the enhancement application for operation on the production server. Correct any errors that are found.
  • When done, you should zip your enhancement folder and submit the zip file to the code submission dropbox in this week's module.

Grading Matrix

  • The controller and view(s) have access to the session: 2 points in objective 2
  • The session is persistent and limited to 2 weeks: 10 points in objective 2
  • Data from the input form is stored in the session: 5 points in objective 2
  • The input form fields are populated from the session data (if any) when loaded: 4 points in objective 2
  • The application continues to work as expected: 5 points in objective 2

Enhancement Value

  • Objective 2: 26 points
