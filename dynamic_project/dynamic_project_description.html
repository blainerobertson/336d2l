Dynamic Project

Project Requirements

The dynamic project is the major project of the semester and is representative of all topics and objectives of the course. Use of M-V-C architecture of you own creation is required (frameworks created by others and use of exercises, enhancements or exam code from the textbook are not allowed). The project must include all of the following:

  • At least one PHP controller for logic processing;
  • At least one PHP model for database interactions;
  • Valid and semantic HTML5 views styled with valid CSS3;
  • Use custom functions to facilitate common code processes;
  • Use all four CRUD data processes involving the database;
  • Incorporate server and client-side data validation to protect the database and insure correct data submission;
  • Use the PDO library and prepared statements for database interactions within the model;
  • Use a database of your own design and construction for data storage.

Project Proposal

The scope and functionality of the project will be determined by you and described in an application proposal. The project must demonstrate your mastery of building a dynamic web-based application. The proposal must thoroughly describe the application to be built, including:

  1. what the application will do;
  2. how the application meets the requirements listed above;
  3. a list and description of each controller, model and view.
  4. provide entity diagrams (see Ch. 16 or https://www.youtube.com/watch?v=-fQ-bRllhXc) of the database(s), tables and fields to be used.
  5. be typed, double-spaced, in a word processing document.
  6. be submitted to the professor during the 8th week of the semester.

Professor response will be in one of the following forms:

  1. Approved – you may proceed with the application project as proposed.
  2. Modify – you must make modifications to the proposal as indicated and resubmit the modified proposal to the professor.
  3. Denied – the project does not meet the requirements or is too large or small of scope. You must submit a new proposal.

Project Video

The finished project is due as specified in the class documentation by being posted to your website and obviously and easily found on the site. In addition, you must post a video demonstrating the use of the project. The video must accomplish three tasks:

  1. How the project application operates
  2. How the application meets the project requirements, including an overview of the code written
  3. How the application fulfills you project proposal

Video Software

You may use any video tool you choose, but several free tools for screencast captures are:

  • Screencast-O-Matic - 15 minute movie length; mp4, FLV or AVI format (recommended)
  • Google Hangouts - Saved to YouTube
  • Quicktime Player - Screencasts or movie capture
