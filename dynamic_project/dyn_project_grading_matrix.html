Dynamic Project Grading Matrix

Introduction

In order to carry out the dynamic project grading, the grader will need access to:

  1. the source code for the application,
  2. the database or source code for the database used by the application,
  3. a web browser with the developer toolbar installed,
  4. the application and video overview delivered from the developer’s production server.

Each objective is worth 25 points with standards and tasks (with values) listed below.

Objective 1

  • Standard: Views are valid and semantic HTML5
    ______ Task: Randomly select 1 view in the public area and 1 view behind the credential wall. Check for semantic structures and markup. Validate for HTML5 compliance. Subtract 1 point per missing semantic structure and 1 point for each validation error to a total of 10 points maximum.
  • Standard: Views are styled using valid CSS3
    ______ Task: Using the same 2 random views for checking the HTML, validate the CSS. Subtract 1 point for each error up to a total of 10 points maximum. Minus 10 automatically if no CSS is used. There is an expectation that CSS is used extensively and not minimally. If the CSS is deemed minimal, if shall be an automatic deduction of 8 points.
  • Standard: Views are usable and provide a consistent user experience.
    ______ Task: As the project views are navigated ask yourself: 1) Are the views consistent in appearance? If Yes, 2 points; if no, 0 points. 2) Is the content easily read (large, clear fonts) and is there good contrast to allow the eye to distinguish between foreground and background colors? If Yes, 3 points; if no, 0 points.

Objective 2

  • Standard: Control structures are well implemented to provide a clear flow control for the application.
    ______ Task: Control structures are used throughout controllers to control the operation of the application. There are no logic operations outside of control structures. 5 points if yes, 0 if no.
  • Standard: Incoming and outgoing data are checked and appropriately delivered to the next step in the application.
    ______ Task: Examine the code of the controller, are control structures in place and do they appropriately check for key/value pairs to trigger application flow and deliver the action appropriate to the trigger? 5 points if yes, 0 if no.
  • Standard: Custom PHP functions are used to provide modular and reusable code.
    ______ Task: Check for use of custom functions to reduce the amount of code and allow for reuse of code for common tasks. 5 points if yes, 0 if no.
  • Standard: The application works.
    ______ Task: The application is fully functional. 10 points if yes, 0 if no.

Objective 3

  • Standard: Models control all database interactions
    ______ Task: Check the model(s), controller(s) and view(s) of the application. Are all database functions contained solely within the model(s). 10 points if yes, 0 if no.
  • Standard: Controllers contain the logic layer and dictate the operational flow of the application
    ______ Task: Review controller(s) for logical control: 1) receiving input from views, 2) as needed calling data via database interaction functions, building logical response messages and delivering views. Do all these operations occur solely in the controller(s)? 5 points if yes, 0 if no.
  • Standard: Views are reserved to interaction / presentation only
    ______ Task: Review the code for views in the application to insure that database functions are not called within views nor should data be submitted back to the view for processing, instead sending all data to a controller. 10 points if yes, 0 if no.

Objective 4

  • Standard: A relational database of user design and creation is used.
    ______ Task: Check the project database. Does it appear to be designed by the project developer? 5 points if yes, 0 if no.
    ______ Task: Is the database being used for all data storage for the project? 5 points if yes, 0 if no.
  • Standard: Data integrity is protected between tables.
    ______ Task: Check for the presence of primary and foreign keys and their use in the database tables. Are they present and used? 5 points if yes, 0 if no.
  • Standard: SQL queries facilitate data interactions and represent a complete CRUD implementation.
    ______ Task: Check the operation of the application and the database functions (in the models). Can data INSERT, SELECT, UPDATE and DELETE be performed by the application? 10 points if yes, 0 if no. (All or nothing – All 4 can be performed and are operational)

Objective 5

  • Standard: Client-side and server-side validation of all inputs is provided.
    ______ Task: Attempt entering incorrect inputs to forms. Does there appear to be client-side validation in place? (Client-side can be in the form of JavaScript or native HTML5 validation) 5 points if yes, 0 if no.
    ______ Task: If the client-side validation appeared to be JavaScript: disable JavaScript and refresh the view; if the client-side validation appeared to be HTML5 validation: go to the same place in the application in Safari (or ask a Mac user to do so for you). Enter the data with intentional errors and submit. Does there appear to be server-side validation? 5 points if yes, 0 if no.
  • Standard: Proper data types are checked and both sanitization and validation are used as applicable.
    ______ Task: Check controllers for use of input filtering/sanitizing/validation for all incoming data (this could be done in a variety of ways). Are these methods being used? 5 points if yes, 0 if no.
  • Standard: Error correction is pushed back to the user for correction.
    ______ Task: When errors are entered in the application, are messages indicating the type of error displayed and the user forced to correct errors before the application will proceed? 5 points if yes, 0 if no.
    ______ Task: When errors are corrected does the application proceed and work correctly? 5 points if yes, 0 if no.

Objective 6

  • Standard: The project and accompanying video are complete and delivered on time.
    ______ Task: The video overview of the application is obvious and can be viewed in the browser. 5 points if yes, 0 if no.
    ______ Task: The video provides a clear overview of the purpose for and how the application functions. 5 points if yes, 0 if no.
    ______ Task: The application functions as demonstrated in the video. 5 points if yes, 0 if no.
  • Standard: Communication within the application is audience appropriate and error free.
    ______ Task: As the application is being used watch for spelling and grammar errors. Subtract 1 point for each error up to a maximum of 5 points.
    ______ Task: Are directions and content delivered in the application clear and audience appropriate? Subtract 1 point for each error up to a maximum of 5 points.
